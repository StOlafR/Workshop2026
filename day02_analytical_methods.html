3 Methods of Analyses

3.1 T Tests

  • One Sample T-Test
  • Two Sample T-Test

For a one-sample t-test we might ask:

Is the average body mass of penguins in 2008 different from the average body mass in 2007?

The code below produces a table of summary statistics (i.e., sample size, mean, median, standard deviation, etc.) of body mass by year.

favstats(body_mass_g ~ year, data = penguins)

2 2008 2700 3612.5   4200 4837.5 6000 4266.667 789.4679 114       0
3 2009 2900 3500.0   4000 4850.0 6000 4210.294 822.9070 119       1

We can also create a box plot that will visualize these differences.

penguins |> # Specify the data set

    !is.na(body_mass_g)  # Filter rows where body_mass_g is not NA
    ) |>
  ggplot(aes(
    x = body_mass_g,
    fill = factor(year) # Make year a factor variable to group box plots
             )
         ) + 
  geom_boxplot() + # Create a box plot layer
  labs(
    title = "Body Mass of Penguins by Year", # Add a title to my plot
    x = "Body Mass (g)", # Rename x axis
    fill = "Year" # Rename legend title
    ) +

We can conduct a one sample t-test using the t.test function

In the t.test function there are three inputs.

  1. The first is a formula to specify that we are only considering a single variable, body_mass_g, which is on the right hand side of the equation.
  2. Second, we specify the data set so R knows where to look for the variable.
  3. And third, we specify a null value to conduct the hypothesis test.
# Create a version of the penguins data set that only contains data from 2008
penguins_008 <- penguins |> 
  filter(year == 2008) 

t.test(~ body_mass_g, data = penguins_008, mu = 4124.541)

  1. What is the p-value? 0.0571

  2. What is the 95% confidence interval? [4120.177, 4413.156]

  3. What is the sample mean of body mass? 4124.541

  4. What can we conclude about the average penguin body mass in 2008? The average body mass of penguins in 2008 is not all that different from 2007.

3.2 Chi Squared Test

Contingency tables or two-way tables are one way of displaying the number of counts between two categorical variables.

While there are a variety of methods for analyzing categorical data - we will briefly cover the chi-squared test of independence. This statistical test is used when the inference of interest is whether the categories of one variable are independent of another.

For example, if we ask the question:

Does the number of penguins on an island differ by year?

As a note, conducting a chi-squared test in R can be a bit tedious however there are alternative packages (see: psych) that are perhaps better suited to more advanced or nuanced applications of the chi-squared test.

To conduct a chi-squared test:

First, create a table() of two categorical variables. Then, pipe the table output to the chisq.test() function.

  1. What variables do we need to specify in the table function?

Type them inside the parenthesis using the data_name$variable_name format.

table(penguins$year, penguins$island)
      
       Biscoe Dream Torgersen
  2007     44    46        20
  2008     64    34        16
  2009     60    44        16
  1. Try piping the table output to the chisq.test() function.
table(penguins$year, penguins$island) |> 
  chisq.test()

    Pearson's Chi-squared test

data:  table(penguins$year, penguins$island)
X-squared = 6.3153, df = 4, p-value = 0.1768
  1. What conclusion can we make? Use the output to support your conclusion.

Year is not independent of island.

3.3 Analysis of Variance (ANOVA)

If we have a numeric variable that is grouped by or explained by a categorical variable, and we want to know whether the means differ between groups, we can conduct an ANOVA.

Once again, let us go back to the penguins data set. This time we will use island as an explanatory variable and bill depth as a response variable to answer the following question:

Does the average bill depth length of penguins differ by island?

3.3.2 Running ANOVA in R

When we are ready to conduct ANOVA to answer our research question, we can use the following code to do that:

aov(bill_depth_mm ~ island, data = penguins)
Call:
   aov(formula = bill_depth_mm ~ island, data = penguins)

Terms:
                  island Residuals
Sum of Squares  531.9082  797.9263
Deg. of Freedom        2       339

Residual standard error: 1.534198
Estimated effects may be unbalanced
2 observations deleted due to missingness

Notice that we use a similar formula notation for specifying the inputs of the ANOVA. Remember that we have bill_depth_mm as our response variable and island as an explanatory variable. Much like a formula or equation we are specifying a left hand side and a right hand side of the equation, and then adding an argument for the data set we are using.

Left Hand Side ~ Right Hand Side 
Response Variable ~ Explanatory Variable

The immediate output is not incredibly informative. It does not contain information about the statistical significance of the ANOVA results.

Often times it is more helpful to assign the model or statistical test as an object to our environment, and then use a function like summary() to get more detailed output.

Tip

The <- is used to assign code to an object. For example, x <- 5 + 2 assigns the sum of 5 and 2 to an object, x.

bill_anova <- aov(bill_depth_mm ~ island, data = penguins) # assign the ANOVA results to an object, bill_anova, to call upon later.

summary(bill_anova) # Extract more detailed results of the ANOVA including
             Df Sum Sq Mean Sq F value Pr(>F)    
island        2  531.9  265.95     113 <2e-16 ***
Residuals   339  797.9    2.35                   
---
Signif. codes:  0 '***' 0.001 '**' 0.01 '*' 0.05 '.' 0.1 ' ' 1
2 observations deleted due to missingness
  1. What conclusion can you make about the different group means? At least one of the island’s mean bill depth is different and statistically significant from the other islands.

If we want to identify which group mean is significantly different, we can use Tukey’s Honest Significant Difference test to evaluate which pair of groups is different. This functions requires us to call up our ANOVA results we saved earlier:

TukeyHSD(bill_anova) # Run a Tukey test using the ANOVA results

    95% family-wise confidence level

  1. Based on the TukeyHSD, which groups are significantly different? The Dream-Biscoe and Torgersen-Biscoe pairs are significantly different.

  2. Which pair has the largest mean difference? The largest difference is in the Torgersen-Biscoe pair.

  3. What is the 95% confidence interval for the Torgersen-Dream pair? The 95% CI is [-.515, .685].

3.4.1 Correlations

Conducting correlations using the base R function is similar to how we make a table.

For example, a correlation between two variables this can be done doing the following:

stats::cor(penguins$bill_depth_mm, penguins$bill_length_mm, use = "complete.obs")
[1] -0.2350529

Notice that when we have NA values we need to tell the correlation function that we want it to use only the rows that have complete data.

The mosaic package also has a function for running correlations that is set up similarly to the favstats function.

mosaic::cor(bill_depth_mm ~ bill_length_mm, data = penguins, use = "complete.obs")
[1] -0.2350529

Correlations for multiple variables is a bit more involved, but can be done!

penguins |> # specify data
  drop_na() |> # drop missing values for simplicity

  cor() # pipes data into the correlation function

3.4.2 Simple Linear Regression

If I want to evaluate how one variable is associated with another, I can conduct a linear regression. For example I might ask the question:

Do penguins with bigger bill depths have a large body mass?

Running a regression model is similar to the way we did ANOVA however this time we will use the lm() function.

Note

You may notice that the formula for lm includes a \(1\) before the explanatory variable. This is a way of explicitly writing out an intercept term but is not necessary to successfully conduct regression.

3.4.3 Evaluating Regression Assumptions

We can also plot the results of regression and create plots to assess the model assumptions.

For linear regression we assume:

  1. Linear relationship between the response and explanatory variables.
  2. Independence of observations.
  3. Normally distributed residuals.
  4. Equal variance of residuals.

There are several ways to look evaluate the regression assumptions of your model. However, we can easily evaluate the conditions for linear regression using the function resid_panel from the ggResidpanel package, which conveniently combines all the plots we can use to visually evaluate the regression assumptions.

library(ggResidpanel) # Need this package to use the resid_panel function
resid_panel(fit1)

4.2.1 Displaying Multiple Models

When you have multiple models that you want to display there are a couple more steps. First, we can assign multiple models into a list. Then, we can put the list of models into either function.

# First, assign regression models to object
fit1 <- lm(body_mass_g ~ 1 + bill_depth_mm, data = penguins)

fit2 <- lm(body_mass_g ~ 1 + bill_depth_mm + flipper_length_mm + species, data = penguins)

fit3 <- lm(body_mass_g ~ 1 +  bill_depth_mm  + flipper_length_mm + bill_depth_mm*species, data = penguins)

# Combine objects into a list and assign it the name models 
models <- list(fit1, fit2, fit3) # 

# Call the object into the function modelsummary
modelsummary(models)

4.3 Creating Figures and Plots with Regression Output

If you want to display the output from regression, you can use the tidy() function to create a tidy tibble of regression and plot them using ggplot. Using the models from Table 1 as an example:

# Create objects of tidy data from regression models, specify that we want confidence intervals
fit1_tidy <- tidy(fit1, conf.int = TRUE)
fit2_tidy <- tidy(fit2, conf.int = TRUE)
fit3_tidy <- tidy(fit3, conf.int = TRUE)

# Since the tidy data all contain the same number and names of columns, we can join them by binding their rows to make a complete data set. 
## The .id argument provides an id column to link each row to its original data set.

fit_plot <- bind_rows(fit1_tidy, fit2_tidy, fit3_tidy, .id = "fit")

Now that we have a tidy data set of our regression output, we can plot the regression coefficient along with error bars.

For example, I may want to see how the slope for bill_depth_mm differs across models.

# Here I am filtering the data to show rows where my term is the slope of interest, then removing columns that I do not want to display in my table, lastly passing the transformed data frame to the kable function to make a table. 
fit_plot |> 
  filter(term == "bill_depth_mm") |> 
  select(-c(std.error, statistic, p.value)) |> 
  kable()

1 bill_depth_mm -191.6428 -229.8353 -153.4502
2 bill_depth_mm 182.3637 146.2523 218.4751

fit_plot |>
